## 1) Concrete system shape

### Control plane

Use **LangGraph** as the outer runtime because it gives you explicit shared state, graph transitions, persistence/checkpointing, and a documented evaluator-optimizer workflow. That is a better fit than unconstrained chat loops when your termination condition is “output validates against a template([LangChain Docs][1])

### Retrieval plane

Split retrieval into two disjoint channels.

**A. Web / current information**

* Tavily Search for discovery
* Tavily Extract or raw page fetch for full-text grounding

Tavily’s docs explicitly position it as search for AI agents / RAG workflows, with separate search and extraction endpoints. ([Tavily Docs][2]). Scientific literature**

* Semantic Scholar Academic Graph API for paper metadata, citations, related-paper expansion
* arXiv API for preprints and metadata

Semantic Scholar exposes paper/author/citation APIs; arXiv exposes programmatic metadata/search access via its API. ([Semantic Scholar][3]) Contract plane
Use **Pydantic** as the output contract:

* define the final schema
* generate JSON schema if needed
* validate every draft output before acceptance

Pydantic v2 supports JSON schema generation and typed validation directly. ([Pydantic Docs][4])

## 2) Reference flow

```text
User template + variables
        ↓
[Normalize task]
        ↓
[Plan research]
  - derive subquestions
  - derive search intents
  - derive evidence requirements
        ↓
[Research loop]
  - web search
  - paper search
  - fetch/extract evidence
  - dedupe + score evidence
  - write draft
        ↓
[Evaluate]
  - schema valid?
  - all required fields populated?
  - citations/evidence threshold met?
  - unresolved claims remaining?
        ↓
[Refine?]
  yes ──> generate missing evidence queries ──> research loop
  no  ──> emit final structured result
```

That is the correct decomposition for your use case: **workflow outside, agentic behavior inside**. LangGraph’s docs explicitly distinguish fixed workflows from dynamic agents, and document evaluator-optimizer loops for iterative quality improvement. ([LangChain Docs][5])

## 3) State model

Use a single graph state object. Keep it small and explicit.

```python
from typing import Literal
from pydantic import BaseModel, Field
from typing_extensions import TypedDict

class InputVars(BaseModel):
    company: str
    topic: str
    region: str | None = None

class EvidenceItem(BaseModel):
    source_type: Literal["web", "paper"]
    title: str
    url: str
    snippet: str
    score: float = 0.0
    supports: list[str] = Field(default_factory=list)

class FinalReport(BaseModel):
    summary: str
    key_findings: list[str]
    scientific_evidence: list[str]
    market_context: list[str]
    open_questions: list[str]
    confidence: Literal["low", "medium", "high"]

class AgentState(TypedDict):
    task_prompt: str
    input_vars: dict
    target_schema_json: dict
    subquestions: list[str]
    web_queries: list[str]
    paper_queries: list[str]
    evidence: list[dict]
    draft_output: dict | None
    validation_errors: list[str]
    missing_requirements: list[str]
    iteration: int
    done: bool
```

Design rule:

* `evidence` stores only normalized evidence objects
* `draft_output` is always schema-shaped JSON, even when invalid
* evaluator never reads raw model prose, only structured draft + evidence

That keeps retries surgical.

## 4) Node graph

This is the concrete node set I would implement.

### Node 1: `plan_task`

Input:

* user script
* variables
* output schema

Output:

* decomposed subquestions
* web queries
* paper queries
* evidence requirements

The planner prompt should produce strictly structured JSON, not prose.

### Node 2: `run_web_research`

For each web query:

* call Tavily Search
* optionally Extract top N URLs
* normalize into `EvidenceItem`

### Node 3: `run_paper_research`

For each paper query:

* query Semantic Scholar
* query arXiv
* normalize into `EvidenceItem`

### Node 4: `rank_and_dedupe_evidence`

Heuristics:

* exact URL dedupe
* title similarity dedupe
* prefer primary sources
* score by recency, source authority, query match

### Node 5: `draft_answer`

LLM writes a draft that must conform to the target schema.

### Node 6: `evaluate_draft`

Checks:

1. Pydantic schema validity
2. required fields not empty
3. each major claim backed by at least one evidence item
4. missing / weak sections identified
5. stop or continue

### Node 7: `generate_gap_queries`

Only when evaluation fails:

* generate the minimum additional queries needed to fill missing fields

### Node 8: `finish`

Emit validated JSON.

## 5) Minimal LangGraph skeleton

This is the right level of concreteness for a first implementation.

```python
import os
import json
import requests
from typing import Any
from pydantic import BaseModel, ValidationError
from langgraph.graph import StateGraph, END

# ---------- Contracts ----------

class FinalReport(BaseModel):
    summary: str
    key_findings: list[str]
    scientific_evidence: list[str]
    market_context: list[str]
    open_questions: list[str]
    confidence: str

# ---------- Tool functions ----------

TAVILY_API_KEY = os.environ["TAVILY_API_KEY"]
S2_API_KEY = os.environ.get("SEMANTIC_SCHOLAR_API_KEY")

def tavily_search(query: str, max_results: int = 5) -> list[dict[str, Any]]:
    resp = requests.post(
        "https://api.tavily.com/search",
        headers={"Authorization": f"Bearer {TAVILY_API_KEY}"},
        json={
            "query": query,
            "max_results": max_results,
            "search_depth": "advanced",
            "include_raw_content": False,
        },
        timeout=30,
    )
    resp.raise_for_status()
    data = resp.json()
    return data.get("results", [])

def semantic_scholar_search(query: str, limit: int = 5) -> list[dict[str, Any]]:
    headers = {}
    if S2_API_KEY:
        headers["x-api-key"] = S2_API_KEY
    resp = requests.get(
        "https://api.semanticscholar.org/graph/v1/paper/search",
        headers=headers,
        params={
            "query": query,
            "limit": limit,
            "fields": "title,year,abstract,url,venue,citationCount,authors"
        },
        timeout=30,
    )
    resp.raise_for_status()
    return resp.json().get("data", [])

def arxiv_search(query: str, start: int = 0, max_results: int = 5) -> list[dict[str, Any]]:
    import feedparser
    url = (
        "http://export.arxiv.org/api/query"
        f"?search_query=all:{query}&start={start}&max_results={max_results}"
    )
    feed = feedparser.parse(url)
    results = []
    for entry in feed.entries:
        results.append({
            "title": entry.title,
            "url": entry.link,
            "summary": entry.summary,
            "published": entry.published,
        })
    return results

# ---------- LLM hooks ----------
# Replace these with your model client. Keep IO strictly JSON.

def llm_json(system: str, user: str, schema: dict | None = None) -> dict[str, Any]:
    raise NotImplementedError("Bind to your model provider")

# ---------- Graph nodes ----------

def plan_task(state: dict) -> dict:
    schema = state["target_schema_json"]
    prompt = {
        "task_prompt": state["task_prompt"],
        "input_vars": state["input_vars"],
        "target_schema": schema,
    }
    out = llm_json(
        system="Decompose the task into research subquestions and retrieval queries. Output JSON only.",
        user=json.dumps(prompt),
    )
    return {
        "subquestions": out["subquestions"],
        "web_queries": out["web_queries"],
        "paper_queries": out["paper_queries"],
        "missing_requirements": [],
    }

def run_web_research(state: dict) -> dict:
    evidence = list(state.get("evidence", []))
    for q in state.get("web_queries", []):
        for r in tavily_search(q, max_results=5):
            evidence.append({
                "source_type": "web",
                "title": r.get("title", ""),
                "url": r.get("url", ""),
                "snippet": r.get("content", "")[:1000],
                "score": 0.0,
                "supports": [],
            })
    return {"evidence": evidence}

def run_paper_research(state: dict) -> dict:
    evidence = list(state.get("evidence", []))
    for q in state.get("paper_queries", []):
        for p in semantic_scholar_search(q, limit=5):
            evidence.append({
                "source_type": "paper",
                "title": p.get("title", ""),
                "url": p.get("url", ""),
                "snippet": p.get("abstract", "") or "",
                "score": float(p.get("citationCount", 0)),
                "supports": [],
            })
        for p in arxiv_search(q, max_results=3):
            evidence.append({
                "source_type": "paper",
                "title": p["title"],
                "url": p["url"],
                "snippet": p["summary"][:1000],
                "score": 0.0,
                "supports": [],
            })
    return {"evidence": evidence}

def rank_and_dedupe_evidence(state: dict) -> dict:
    seen = set()
    ranked = []
    for item in state.get("evidence", []):
        key = item["url"] or item["title"].strip().lower()
        if key in seen:
            continue
        seen.add(key)
        ranked.append(item)

    def rank_key(x: dict):
        primary_bonus = 1 if x["source_type"] == "paper" else 0
        return (primary_bonus, x.get("score", 0.0), len(x.get("snippet", "")))

    ranked.sort(key=rank_key, reverse=True)
    return {"evidence": ranked[:30]}

def draft_answer(state: dict) -> dict:
    prompt = {
        "task_prompt": state["task_prompt"],
        "input_vars": state["input_vars"],
        "subquestions": state["subquestions"],
        "evidence": state["evidence"][:20],
        "output_schema": state["target_schema_json"],
    }
    out = llm_json(
        system=(
            "Produce the final answer as JSON only. "
            "Use only supported claims from evidence. "
            "Leave uncertain items in open_questions."
        ),
        user=json.dumps(prompt),
        schema=state["target_schema_json"],
    )
    return {"draft_output": out}

def evaluate_draft(state: dict) -> dict:
    errors = []
    missing = []

    try:
        report = FinalReport.model_validate(state["draft_output"])
    except ValidationError as e:
        return {
            "validation_errors": [str(e)],
            "missing_requirements": ["schema_validation_failed"],
            "done": False,
        }

    if not report.summary.strip():
        missing.append("summary")
    if not report.key_findings:
        missing.append("key_findings")
    if not report.scientific_evidence:
        missing.append("scientific_evidence")
    if not report.market_context:
        missing.append("market_context")

    done = len(missing) == 0
    return {
        "validation_errors": errors,
        "missing_requirements": missing,
        "done": done,
    }

def generate_gap_queries(state: dict) -> dict:
    if state["done"]:
        return {}

    prompt = {
        "missing_requirements": state["missing_requirements"],
        "existing_subquestions": state["subquestions"],
        "existing_evidence_titles": [e["title"] for e in state["evidence"][:20]],
        "task_prompt": state["task_prompt"],
        "input_vars": state["input_vars"],
    }
    out = llm_json(
        system=(
            "Generate the minimum extra web_queries and paper_queries needed "
            "to fill the missing requirements. Output JSON only."
        ),
        user=json.dumps(prompt),
    )

    return {
        "web_queries": state["web_queries"] + out.get("web_queries", []),
        "paper_queries": state["paper_queries"] + out.get("paper_queries", []),
        "iteration": state["iteration"] + 1,
    }

def route_after_eval(state: dict) -> str:
    if state["done"] or state["iteration"] >= 3:
        return "finish"
    return "gap_queries"

def finish(state: dict) -> dict:
    return state

# ---------- Build graph ----------

def build_graph():
    g = StateGraph(dict)
    g.add_node("plan", plan_task)
    g.add_node("web", run_web_research)
    g.add_node("papers", run_paper_research)
    g.add_node("rank", rank_and_dedupe_evidence)
    g.add_node("draft", draft_answer)
    g.add_node("eval", evaluate_draft)
    g.add_node("gap_queries", generate_gap_queries)
    g.add_node("finish", finish)

    g.set_entry_point("plan")
    g.add_edge("plan", "web")
    g.add_edge("web", "papers")
    g.add_edge("papers", "rank")
    g.add_edge("rank", "draft")
    g.add_edge("draft", "eval")
    g.add_conditional_edges("eval", route_after_eval, {
        "gap_queries": "gap_queries",
        "finish": "finish",
    })
    g.add_edge("gap_queries", "web")
    g.add_edge("finish", END)

    return g.compile()
```

This uses exactly the pattern the LangGraph docs support: explicit state, graph nodes, and evaluator-optimizer iteration. ([LangChain Docs][1])

## 6) The prompt contracts

Do not let the model “decide the whole workflow.” Give it bounded jobs.

### Planner prompt

Input:

* task script
* variables
* target schema

Output JSON:

```json
{
  "subquestions": ["..."],
  "web_queries": ["..."],
  "paper_queries": ["..."]
}
```

### Draft prompt

Input:

* normalized evidence
* schema

Output:

* JSON matching the final schema only

### Gap-query prompt

Input:

* validation failures
* missing fields
* current evidence

Output JSON:

```json
{
  "web_queries": ["..."],
  "paper_queries": ["..."]
}
```

This is the core reliability move: every model call has a narrow contract.

## 7) Why this works better than “one autonomous research agent”

Because your actual requirement is not “autonomy.” It is:

1. satisfy a structured output contract
2. gather enough external evidence
3. stop only when the contract is met

That is not a chat problem. It is a **bounded iterative optimization problem**. LangGraph’s workflow/agent distinction and evaluator-optimizer pattern line up with that directly. ([LangChain Docs][5])

## 9) Production hardening

The skeleton above is correct structurally, but for production you need four additional controls.

### A. Evidence provenance

Store:

* query that found the source
* retrieval timestamp
* source authority score
* claim mapping

Otherwise your evaluator cannot distinguish “filled” from “supported.”

### B. Budget / loop caps

Hard caps:

* max iterations
* max searches per iteration
* max evidence items
* max token budget per draft

### C. Source-class policy

For each output field, define allowable evidence classes.

Example:

* `scientific_evidence` → papers only
* `market_context` → web or industry reports
* `summary` → any supported evidence

### D. Deterministic acceptance

Acceptance should be code, not model judgment:

* schema valid
* no required field empty
* minimum evidence count per field
* no unsupported claims markers

## 11) Concrete recommendation

If I were implementing this for an engineering org, I would start with:

* **LangGraph** for orchestration
* **Tavily** for web discovery/extraction
* **Semantic Scholar + arXiv** for paper retrieval
* **Pydantic** for schema enforcement
* plain `requests` for tools
* no vector DB in v1
* no multi-agent roleplay in v1

That gets you the shortest path to a system that is:

* inspectable
* retryable
* template-constrained
* incrementally hardenable

The one thing I would not do is start from an unconstrained “research agent” product and hope it naturally converges to your output schema.
